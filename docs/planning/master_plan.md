# Master Plan

## 最終ゴール

**SSIM / PSNR において上位入賞を達成する**

達成条件:
- SSIM ≥ 0.95（暫定目標、ベースライン計測後に調整）
- PSNR ≥ 35 dB（暫定目標、ベースライン計測後に調整）

---

## 開発思想

### 原則1: 仮説駆動

```
仮説 → 実験 → 計測 → 判断 → 次の仮説
```

このサイクルを**1日3回以上**回すことが目標。

### 原則2: CI = 判断エンジン

- CIは「コードが動くか」を確認する場所ではない
- CIは「この変更は改善か？」を判断する装置

### 原則3: 記録なき実験は存在しない

- すべての実験は `experiments/` に記録
- 結果が空の実験ディレクトリは不正

### 原則4: mainは知見の集積地

- mainブランチには「改善が確認されたもの」のみマージ
- 実験段階のコードはworktree / featureブランチで作業

---

## やらないこと（明示的な除外）

| やらないこと | 理由 |
|--------------|------|
| 手動での評価指標計算 | CI自動化で代替 |
| 仮説なしの「とりあえず試す」 | 学習が蓄積されない |
| 結果を記録しない実験 | 存在しないのと同じ |
| mainブランチでの直接開発 | 並列実験を阻害 |
| 過度なハイパーパラメータ探索 | アーキテクチャ改善を優先 |
| 複数仮説の同時検証 | 因果関係が不明になる |

---

## 成功の定義

1. **実験速度**: 1日3実験以上を維持
2. **知見蓄積**: 各実験から明確な学びが記録される
3. **判断速度**: CI結果から30分以内に次の仮説が立つ
4. **コード品質**: すべてのテストがパスする状態を維持
